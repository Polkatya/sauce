# Деплой на Cloudflare Pages

## Структура проекта
```
/index.html          — страница-генератор ссылок
/functions/view.js   — Cloudflare Pages Function, рендерит страницу с видео по ?v=&i=&t=&s=
```

`netlify.toml` больше не нужен — удали его, Cloudflare сам находит `/functions/*.js`.

## Как задеплоить

### Вариант A — через сайт Cloudflare (без консоли)
1. Зайди на https://dash.cloudflare.com → Workers & Pages → **Create application** → **Pages** → **Upload assets** (или Direct Upload).
2. Перетащи туда всю папку **целиком** (и `index.html`, и папку `functions` вместе с `view.js` внутри) — важно сохранить структуру.
3. Cloudflare сам определит `/functions/view.js` как функцию на маршруте `/view`.
4. После деплоя получишь адрес вида `https://твой-проект.pages.dev`.

### Вариант B — через Git (GitHub/GitLab), рекомендуется
1. Залей проект в репозиторий с такой же структурой (index.html в корне, functions/view.js).
2. В Cloudflare Pages → **Connect to Git** → выбери репозиторий.
3. Build command — оставь пустым, Output directory — `/` (корень).
4. Deploy. При каждом пуше в репозиторий сайт обновится сам.

## Как пользоваться после деплоя
1. Открываешь `https://твой-проект.pages.dev/` (генератор) — с телефона или ПК.
2. Вставляешь прямую ссылку на MP4 и картинку-превью, название, таймер.
3. Жмёшь «Создать ссылку» — получаешь `https://твой-проект.pages.dev/view?v=...&i=...&t=...&s=...`.
4. Эту ссылку кидаешь в Discord — никаких файлов заливать не нужно, страница рендерится на лету.

## Важно про ссылки на видео
Discord CDN-ссылки (`cdn.discordapp.com/attachments/...` и `media.discordapp.net/...`) **временные** — у них есть параметр `ex=` (expire) и через некоторое время они перестают открываться. Для постоянных ссылок используй сторонний хостинг файлов (например file.garden, который уже встречается в твоих старых страницах) — такие ссылки не протухают.
